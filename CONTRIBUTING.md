# Contributing to SkillRelay

Thank you for your interest in contributing to SkillRelay!

SkillRelay is currently in the **design and planning stage**. The codebase is not yet implemented, so the most valuable contributions right now are around design, architecture, and documentation.

## How to Contribute

### Reporting Issues

Use the GitHub issue tracker to:

- Report bugs (once implementation begins)
- Propose new features or improvements
- Ask design questions
- Suggest changes to the architecture or CLI design

Please search existing issues before opening a new one.

### Proposing Design Changes

If you have thoughts on the architecture, CLI design, skill format, or adapter interface, open an issue with the `design` label and describe your proposal.

### Submitting Pull Requests

1. Fork the repository
2. Create a feature branch from `main`
3. Make your changes
4. Ensure any documentation is updated to reflect your changes
5. Open a pull request with a clear description of what you changed and why

### Coding Conventions

The implementation language has not yet been decided. Once it is, this section will be updated with language-specific conventions, formatting rules, and tooling requirements.

For now, follow these general principles from the project:

- CLI command name: `skillrelay`
- Brand and class names: PascalCase (`SkillRelay`, `Skill`, `Source`, `Registry`, `Adapter`)
- File and directory names: kebab-case lowercase
- Do not use the old codename "SkillMesh" anywhere
