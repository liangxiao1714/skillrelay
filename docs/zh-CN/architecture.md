# SkillRelay 架构设计

> **状态**：设计阶段 —— 本文档持续更新中。

## 概述

SkillRelay 是一个以 CLI 为核心的本地技能仓库与多 Agent 桥接工具。本机作为统一的中心技能仓库，各 Agent 通过专属适配器处理格式转换和同步。

## 核心组件

### Registry（仓库）

存储在用户本机的本地中心仓库，包含：

- 规范化的技能内容
- 技能元数据（名称、版本、标签、来源、兼容性）
- 来源记录（技能的获取来源）
- 各 Agent 的同步状态

仓库是所有已纳管技能的唯一事实来源。

### Source（来源）

技能的获取来源。计划支持的来源类型：

- **SkillHub** —— 专用的在线技能平台
- **GitHub 仓库** —— 包含技能或技能包的代码仓库
- **本地目录** —— 本机上的本地文件目录
- **Agent 技能目录** —— 其他 Agent 已安装的技能目录
- **单个文件** —— 单个 `SKILL.md` 或类似格式的文件

### Adapter（适配器）

针对特定 Agent 的集成层，每个适配器负责：

- 理解目标 Agent 的技能格式和目录规范
- 将该 Agent 的技能导入仓库
- 将仓库中的技能导出到该 Agent
- 在规范格式与 Agent 专属格式之间进行转换
- 上报同步状态

计划支持的适配器：Hermes、OpenClaw、Claude Code、OpenCode、Codex。

### Skill（技能）

一个可复用的 Agent 知识、工作流或能力单元。规范的技能记录包含：

- **标识信息**：名称、版本、唯一 ID
- **元数据**：描述、标签、分类、作者
- **来源信息**：来源、来源 URL、导入时间
- **兼容性**：支持的 Agent 列表、格式要求
- **内容**：技能的实际定义

## 数据流

```
外部来源                      本地仓库                  Agent
（SkillHub、GitHub、   -->   （本机规范存储）   -->   （Hermes、Claude、
  本地目录……）                                          OpenClaw……）
                                   ^
                                   |
                             适配器负责双向
                             格式转换
```

技能**双向流转**：

- `来源 → 仓库 → Agent`（导入与分发）
- `Agent → 仓库 → 来源`（采集与发布）

## 设计原则

1. **每台机器一个中心仓库** —— 所有已纳管技能均存储于此。
2. **按需同步** —— 技能不会被强制全量加载到所有 Agent。
3. **适配器隔离** —— Agent 只通过各自的适配器与仓库交互。
4. **双向流转** —— 仓库既可消费技能，也可生产和发布技能。
5. **可扩展性** —— 可在不修改核心的前提下添加新的适配器和来源类型。

## CLI 架构

主接口为 CLI 工具，命令为 `skillrelay`。各功能领域：

| 领域 | 计划命令 |
|---|---|
| 来源管理 | `source add`、`source remove`、`source list` |
| 搜索与发现 | `search`、`list` |
| 导入 | `install`、`import` |
| 导出与分发 | `export`、`push` |
| 同步 | `sync`、`pull` |
| 转换 | `convert` |
| 校验 | `check`、`validate` |
| 状态 | `status`、`info` |
| 配置 | `config` |
| 发布 | `publish` |
