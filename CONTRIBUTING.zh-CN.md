# 参与贡献 SkillRelay

感谢你对 SkillRelay 的关注！

SkillRelay 目前处于**设计与规划阶段**，代码尚未实现。因此，目前最有价值的贡献集中在设计、架构和文档方面。

## 如何贡献

### 提交 Issue

使用 GitHub issue 跟踪器来：

- 报告 Bug（待实现阶段开始后）
- 提出新功能或改进建议
- 提出设计问题
- 对架构或 CLI 设计提出修改建议

提交前请先搜索现有 issue，避免重复。

### 提出设计方案

如果你对架构、CLI 设计、技能格式或适配器接口有想法，请带上 `design` 标签开一个 issue，详细描述你的方案。

### 提交 Pull Request

1. Fork 本仓库
2. 从 `main` 分支创建功能分支
3. 完成修改
4. 确保文档已根据修改内容同步更新（包括中英文两个版本）
5. 提交 PR，清楚描述改动内容及原因

### 文档贡献

本项目维护中英文双语文档。修改文档时，请尽量同步更新对应的另一语言版本，或在 PR 中注明需要翻译协助。

### 代码规范

实现语言尚未确定，待确定后本节将补充具体的语言规范、格式要求和工具链说明。

目前请遵循以下通用原则：

- CLI 命令名：`skillrelay`
- 品牌与类名：PascalCase（`SkillRelay`、`Skill`、`Source`、`Registry`、`Adapter`）
- 文件和目录名：kebab-case 小写
- 禁止在任何地方使用旧代号 "SkillMesh"

## 行为准则

请保持尊重、建设性和包容的态度，遵循开源社区通用规范。

向 SkillRelay 提交贡献，即表示你同意你的贡献将以 [MIT 许可证](./LICENSE) 发布。
